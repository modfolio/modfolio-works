# Layout Patterns — 생태계 레이아웃 규격

> **참조용 규격.** 각 앱은 독립 디자인 시스템(House of Brands)을 가진다.
> 이 규격은 강제가 아니라, 전통적인 헤더/푸터를 구축할 때 **구조와 정보 구성**을 참고하기 위한 것이다.
> Reference Implementation: `universe.modfolio.io` (modfolio-universe/apps/dashboard)

## 이 규격이 정의하는 것 vs 하지 않는 것

| 정의함 (구조적 규격) | 정의 안 함 (브랜드 영역) |
|----------------------|--------------------------|
| 헤더 높이 권장값 | 배경색, 블러, 투명도 |
| 로고(좌) + CTA(우) 배치 원칙 | 색상 팔레트, 폰트 |
| 푸터 정보 구성 (상단/하단 분리) | 시각적 스타일, 애니메이션 |
| 푸터에 포함할 필수 정보 | 레이아웃 변형 (가로 스크롤 등) |
| 반응형 브레이크포인트 권장값 | 컴포넌트 구현 방식 |
| 뷰포트 폭 사용 원칙 | CSS 프레임워크 선택 |

## 헤더 규격

### 구조 원칙

```
┌──────────────────────────────────────────────────────┐
│  [로고]              [네비게이션(선택)]         [CTA] │
│  ← 좌측 끝                                  우측 끝 →│
└──────────────────────────────────────────────────────┘
   뷰포트 전폭, padding으로 좌우 여백
```

### 권장값

| 항목 | 값 | 비고 |
|------|-----|------|
| 높이 | `80px` | 72~90px 범위 허용 |
| 위치 | `position: fixed` | 스크롤 시 상단 고정 |
| 좌우 여백 | `clamp(1rem, 4vw, 2rem)` | 반응형 |
| 뷰포트 폭 | **전폭** (max-width 없음) | 로고/CTA가 양 끝에 붙음 |

### 배치 원칙

- **로고**: 항상 좌측 끝
- **CTA/로그인**: 항상 우측 끝
- **네비게이션**: 앱 스타일에 따라 선택
  - 중앙 배치 (universe, connect)
  - 우측 CTA 옆 배치
  - 좌측 로고 옆 배치
  - 햄버거/메가메뉴 안으로 (press 등 비전통적 레이아웃)

### 호버 밑줄 (선택)

네비게이션 링크에 호버 밑줄을 넣는 경우:
- 헤더 하단 border와 같은 위치에 정렬
- 중앙에서 좌우로 동시 펼침 (`transform-origin: center`)

### 비전통적 레이아웃에서의 헤더

가로 스크롤, 풀스크린 등 전통적 헤더가 어울리지 않는 경우:
- 로고 + CTA 배치 원칙은 유지하되 구현 방식은 자유
- 예: 오버레이 네비게이션, 사이드 메뉴, 플로팅 버튼 등

## 푸터 규격

### 구조 원칙: 상단/하단 2단 분리

```
┌──────────────────────────────────────────────────────┐
│  상단 (메인)                                         │
│  ┌─────────┬──────────┬──────────┬──────────┐       │
│  │ 브랜드   │ Products │ Platform │ Ecosystem│       │
│  │ 설명     │ App      │ Connect  │ Works    │       │
│  │ 상태     │ GistCore │ Pay      │ LS       │       │
│  │ SNS      │ ...      │ ...      │ ...      │       │
│  └─────────┴──────────┴──────────┴──────────┘       │
├──────────────────────────────────────────────────────┤
│  하단 (법적 정보)                                     │
│  사업자 정보 · 연락처 (좌측)    © 저작권 (우측)        │
└──────────────────────────────────────────────────────┘
```

### 상단 (메인 영역)

**브랜드 컬럼** — 각 앱의 이름 + 한 줄 설명:
- 서비스 상태 표시 (선택)
- SNS 아이콘: Instagram, LinkedIn, YouTube

**링크 컬럼** — 3개 권장:

| 컬럼 | 포함 항목 |
|------|----------|
| **Products** | App, GistCore, Naviaca, Sincheong, Worthee, Munseo, Press |
| **Platform** | Connect, Pay, Admin, On, Docs |
| **Ecosystem** | Works, LS, Axiom, Studio |

> 각 앱 푸터에서는 자기 자신을 목록에서 빼거나 강조할 수 있음.
> Modfolio 컨텍스트 안이므로 "Modfolio" 접두사는 생략.

### 하단 (법적 정보 바)

**좌측 — 사업자 정보** (중간점 `·` 구분):
```
모드폴리오 · 대표 김동헌 · 사업자등록번호 104-95-65636 · 통신판매업 제2025-수원팔달-0953호 · 출판사 제2024-000037호 · 경기도 수원시 팔달구 효원로 278, 6층 602호 · contact@modfolio.io
```

**우측 — 저작권**:
```
© 2026 Modfolio. All rights reserved.
```

### 비전통적 레이아웃에서의 푸터

가로 스크롤 등 전통적 푸터가 어울리지 않는 경우:
- **정보 구성은 동일하게 유지** (사업자 정보, 링크, 저작권)
- 표현 방식은 자유 (예: 마지막 슬라이드, 오버레이 패널, 축소된 바 등)

## 섹션 레이아웃

### 전폭 배경 + 콘텐츠 센터링 패턴

```
┌─ 뷰포트 전폭 ──────────────────────────────────────┐
│  ┌─ max-width: 1280px, 센터링 ──────────────────┐  │
│  │  콘텐츠                                       │  │
│  └───────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────┘
```

### 권장 토큰

| 토큰 | 값 | 용도 |
|------|-----|------|
| `--mu-max-width` | `1280px` | 콘텐츠 최대 폭 |
| `--mu-padding-x` | `clamp(1rem, 4vw, 2rem)` | 좌우 여백 |

### 반응형 브레이크포인트

| 이름 | 값 | 용도 |
|------|-----|------|
| sm | `640px` | 모바일 |
| md | `768px` | 태블릿 / 모바일 네비게이션 전환 |
| lg | `1024px` | 데스크톱 |
| xl | `1280px` | 와이드 데스크톱 |
